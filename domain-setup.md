# Подключение домена rules.kalloware.space к GitBook

Эта инструкция нужна, чтобы открыть GitBook по адресу:

```text
rules.kalloware.space
```

## 1. Подключить GitHub к GitBook

1. Открой GitBook.
2. Создай новый Space или открой уже созданный.
3. В настройках Space подключи синхронизацию с GitHub.
4. Выбери репозиторий с этими файлами.
5. Укажи ветку `main`.

## 2. Добавить домен в GitBook

1. Открой настройки публикации в GitBook.
2. Найди раздел с custom domain.
3. Добавь домен:

```text
rules.kalloware.space
```

4. GitBook покажет DNS-запись, которую нужно добавить у регистратора домена.

## 3. Добавить DNS-запись

В DNS-зоне домена `kalloware.space` создай запись:

| Тип | Имя | Значение |
| --- | --- | --- |
| CNAME | rules | значение, которое покажет GitBook |

Важно: точное значение CNAME нужно брать из GitBook. Обычно GitBook сам показывает, куда должен указывать домен.

## 4. Дождаться проверки

После добавления DNS-записи вернись в GitBook и нажми проверку домена.

DNS может обновляться не сразу. Обычно это занимает от нескольких минут до нескольких часов, иногда до 48 часов.

## 5. Не менять текущий CNAME сайта

В репозитории уже есть файл `CNAME` для `site.kalloware.space`. Его не нужно менять, если основной сайт должен продолжать работать отдельно.

Для GitBook-домена `rules.kalloware.space` настройка делается через DNS и настройки GitBook.
